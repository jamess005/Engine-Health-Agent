# Engine Health Agent

Predictive maintenance agent for turbofan engines using the NASA CMAPSS FD004 dataset.
Predicts Remaining Useful Life (RUL), diagnoses anomalous degradation, assesses flight
safety, and monitors model drift — all without an LLM.

---

## Overview

The agent uses a **deterministic protocol orchestrator** backed by a ForecastNet ensemble
(multi-scale 1D CNN + squeeze-excitation + multi-head attention) to deliver consistent,
hallucination-free engine health reports.

```
NASA CMAPSS FD004
(248 test engines, 6 operating conditions, 21 sensors)
        │
        ▼
Feature Engineering ──► ForecastNet v5c Ensemble ──► RUL prediction
(normalisation,          (4 seeds, 80% CI)             + confidence interval
 condition clustering,
 rolling statistics)
        │
        ▼
Deterministic Orchestrator
  1. estimate_rul          — ForecastNet prediction + model accuracy tier
  2. compute_degradation_rate — fleet-relative sensor slope comparison
  3. diagnose_engine        — root-cause analysis (conditional on rate)
  4. compare_conditions     — best/worst operating condition
  5. forecast_journey       — per-leg RUL inference
  6. schedule_maintenance   — urgency tier
  7. find_replacement_engine — ranked by true RUL from RUL_FD004.txt
  8. Recommendation         — deterministic template (no LLM)
        │
        ▼
SQLite DB (data/engine_health.db)
  predictions · agent_runs · drift_metrics · raw_sensor_data
```

## Setup

### 1. Clone and create virtual environment

```bash
git clone https://github.com/jamess005/Engine-Health-Agent.git
cd Engine-Health-Agent
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env: set DB_USER to your OS username
```

### 3. Obtain data and models

The following are excluded from version control (see `.gitignore`):

- `data/raw/` — NASA CMAPSS FD004 dataset (download from NASA DASHlink)
- `data/processed/` — pre-processed parquet files (generated by notebooks)
- `models/gru/` — trained ForecastNet checkpoint (`forecast_model.pt`)
- `models/preprocessing/` — scaler, KMeans cluster model, fleet statistics

Run notebooks in order (`01` → `06`) to reproduce preprocessing and training.

## Fleet Health Overview

```bash
python -m src.cli.app --fleet
```

Buckets all 248 engines from the FD004 test set using ground-truth RUL
(from `RUL_FD004.txt`) — no model inference required, runs in ~0.2s:

| Tier | Threshold | Action |
|------|-----------|--------|
| ✗ GROUNDED  | RUL ≤ 6  | Immediate removal from service |
| ⚠ CRITICAL  | RUL 7–10 | Maintenance before next flight |
| ⚠ ADVISORY  | RUL 11–30 | Schedule maintenance soon |
| ✓ HEALTHY   | RUL > 30 | No immediate action |

## Drift Monitoring

```bash
python -m src.cli.app --drift
```

Uses **Population Stability Index (PSI)** to detect distribution shift between the
training baseline and current test fleet sensor readings.

| PSI | Status |
|-----|--------|
| < 0.10 | No drift |
| 0.10–0.25 | Moderate drift — monitor |
| > 0.25 | Significant drift — investigate |

Results are stored in `data/engine_health.db` (`drift_metrics` table). Significant drift
suggests the operating environment has changed from training conditions and model
accuracy may be degraded.

## Dataset

NASA CMAPSS (Commercial Modular Aero-Propulsion System Simulation) FD004:
- **249 training engines**, **248 test engines**
- **21 sensors**, **3 operating condition variables**
- **6 operating conditions** (KMeans-clustered from altitude, Mach, TRA)
- Run-to-failure simulation — each engine degrades until failure
